# 成本控制与自动化

> 第一阶段的目标不是"赚多少钱"，而是"用最小成本验证一个方向"。

## 第一阶段成本原则

| 项目 | 建议 | 大概费用 |
|------|------|---------|
| 域名 | 只买一个，.com 或目标国家域名 | ~$10-15/年 |
| 托管 | 免费静态托管（Cloudflare Pages / Vercel） | $0 |
| 分析工具 | Search Console + GA4 + Google Trends | $0 |
| 关键词工具 | 先用免费的，暂不买付费工具 | $0 |
| 图片处理 | Squoosh / Sharp | $0 |
| AI 辅助 | 用已有的工具（如 Cursor / ChatGPT） | 0（如果你已有） |
| 多域名 | 暂不批量注册 | $0 |

**不要在第一阶段花这些钱：**

- ❌ 多个域名
- ❌ 付费 SEO 工具（Ahrefs、SEMrush）
- ❌ 付费 AI 内容生成工具
- ❌ 高价托管方案
- ❌ 外链购买或 PBN

## 什么时候可以花钱

```txt
你已经验证的项目满足：
✅ 搜索需求真实存在
✅ 你的页面获得自然展示
✅ 你有一个可重复的内容流程
✅ 你确认投入更多时间/资源能放大结果

这时候再考虑：
- 购买第二个域名
- 使用 Ahrefs 付费版做深度关键词分析
- 外包部分重复性工作
```

## 什么时候考虑自动化

> 首先确保你手动能稳定完成流程。

适合自动化的是**重复劳动**：

| 可自动化的 | 不可自动化的 |
|-----------|-------------|
| 创建页面骨架 | 需求判断 |
| 压缩图片 | 事实核验 |
| 批量转换格式 | 用户价值判断 |
| 检测死链 | 版权判断 |
| 生成 Sitemap | 内容质量审核 |
| 汇总数据 | 策略决策 |

### 自动化工具推荐

```bash
# 图片压缩
npx squoosh-cli --webp '{quality:80}' input.png

# 检测死链
npx broken-link-checker https://yoursite.com

# Sitemap 自动生成
# Astro：@astrojs/sitemap（内置）
# Next.js：next-sitemap
```

### 什么时候不适合自动化

```txt
"我在用 AI 每天生成 100 篇文章，看哪个能拿到流量。"
→ 这是规模化内容滥用，Google 明确禁止。

"我先手动做 5 个页面，然后分析哪个方向有需求。
  如果确定方向，再考虑用工具批量辅助。
  但我仍然亲自验证每个数据。"
→ 这是合理的自动化策略。
```

## 时间和预算上限

开始第一个实验项目时，给自己设定：

```txt
时间预算：每天 1-2 小时，30 天
金钱预算：$15（一个域名）
停止条件：
  - 30 天后没有任何正向数据信号
  - 发现自己无法持续产出有验证的内容
  - 成本超出预算
  - 其他更优先的事项出现
```

**"放弃"不是失败。** 如果你在 30 天内学到了"这个方向的需求不成立"或"我自己并不愿意持续做这件事"，这就是有价值的信息。
