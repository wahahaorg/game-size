# 广告接入

> 请读完这段再做广告申请。急不是策略。

## 正确顺序

```
需求成立 → 内容有价值 → 页面被收录
  → 获得相对稳定的自然流量
  → 检查版权、隐私和广告政策
  → 再申请广告
```

不要在没有流量的网站上申请。不要在只有几个页面的时候申请。**不要在没有实质内容的页面上申请。**

## AdSense 申请前检查清单

- [ ] 网站有实质内容（不只堆砌关键词或只有广告空白内容）
- [ ] 每个页面都有一定长度和质量的内容
- [ ] 网站有隐私政策页面
- [ ] 网站有联系方式和清晰导航
- [ ] 网站遵守 Google 内容政策
- [ ] 内容不涉及赌博、成人、毒品、武器、作弊等禁止内容
- [ ] 没有侵犯版权的图片、内容或商标
- [ ] 网站设计合理，移动端正常运行

### 隐私政策

每个网站都需要隐私政策。重点说明：

- 收集哪些用户数据
- 如何使用这些数据
- 是否使用 Cookie（GA4 和 AdSense 都会用）
- 用户如何控制自己的数据

一个简单的开始：

```txt
你的网站隐私政策应至少包含：
- 数据收集方式（Google Analytics、AdSense）
- Cookie 使用说明
- 第三方服务（Google 服务）
- 用户的数据权利
- 联系方式
```

### 联系页面

至少有一个能让用户联系到你的方式，可以是联系表单或公开的邮箱地址。

## 申请流程

```txt
1. 确认网站满足上述条件
2. 打开 https://adsense.google.com
3. 填写网站 URL 和收款信息
4. 提交申请
5. 等待审核（几小时到几周）
6. ✅ 通过 → 把广告代码添加到网站
7. ❌ 拒绝 → 根据拒绝原因修改后重新提交
```

## 广告放置原则

**做对的事情：**

- 把广告放在**不影响内容阅读**的位置
- 控制广告数量，不要过多
- 选择与内容类型匹配的广告格式
- 检查是否在移动端显示正常

**不要做的事情：**

- ❌ 把广告放在游戏按钮或操作控件附近
- ❌ 诱导用户点击广告（"点击这里继续"）
- ❌ 广告伪装成下载按钮或内容入口
- ❌ 只有广告、没有实质内容的页面
- ❌ 在游戏地图或攻略截图上叠加广告

## 风险提醒

| 风险 | 说明 |
|------|------|
| 审核通过 ≠ 永久安全 | Google 随时可能封停账户，原因包括无效流量或政策违规 |
| 审核快 ≠ 没有风险 | 不要误以为审核快就代表安全 |
| 流量来源复杂 | 非自然流量、机器人流量可能导致封号 |
| 税务问题 | 收款涉及国家间税务政策，如果在美国有收入还需 W8-BEN 表格 |
| 结算延迟 | 达到付款门槛后可能还有 30~60 天账期 |

**最保守的策略：先验证内容有价值、搜索能带来自然流量，再接入广告。**
