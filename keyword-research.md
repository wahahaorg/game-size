# 关键词调研

> 很多人一上来就找"搜索量大、竞争低"的词，但真正的关键是：**用户到底想解决什么问题？**

## 调研流程

```
收集候选词 → 逐个分析意图和需求
  → 查看 Google 前十名竞争页面
  → 判断自己能否提供增量价值
  → 决定做、暂缓或放弃
```

## 第一步：从哪里找候选词

初期不需要花钱买工具。免费渠道就够：

- **Google 搜索联想**：在搜索框输入"游戏名 + 攻略/指南/怎么"，看下拉词
- **Google 搜索相关搜索**：拉到搜索结果页底部，看"相关搜索"
- **Google Trends**：对比不同词的搜索趋势，确认需求是否真实
- **Reddit / Discord**：看玩家在问什么问题
- **游戏官方论坛 / Wiki**：内容过时或不完整的部分就是机会
- **Ahrefs 免费工具**：Ahrefs 提供一些免费的关键词分析功能
- **同类型网站**：看排名靠前的攻略站在做什么主题，但不要照抄

## 第二步：分析搜索意图

每个候选关键词，先问这 4 个问题：

| 问题 | 判断标准 |
|------|---------|
| 用户想解决什么问题？ | "Genshin Impact Nahida build" → 想知道怎么搭配圣遗物和武器 |
| 是查询、工具、导航还是购买？ | 决定你做攻略文章、计算器、还是购买入口 |
| 用户在什么场景下搜？ | 玩游戏时用手机查？还是上班时电脑查？影响页面设计 |
| 问题解决后还可能搜什么？ | 决定你的内链结构和扩展方向 |

## 第三步：分析搜索结果（SERP）

打开 Google，搜索你的候选关键词，**看完前十名的页面**。用一个表格记录：

| 排名 | 网站 | 页面类型 | 更新时间 | 优点 | 缺口 |
|------|------|---------|---------|------|------|
| 1 | wikidot.net | 攻略列表 | 2024 | 信息全面 | 排版混乱，没有截图 |
| 2 | reddit.com | 讨论帖 | 2023 | 真实玩家经验 | 信息分散，不系统 |
| 3 | fandom.com | 百科页面 | 2025 | 有数据表 | 加载慢，广告多 |

**分析的目的是找"缺口"——现有页面没有覆盖或做得不够好的地方：**

- 内容是否过时？（版本更新后信息还准确吗？）
- 回答是否不完整？（只说了"选什么"，没解释"为什么"）
- 是否有更好的呈现方式？（文字攻略 → 视频/图片/计算器）
- 是否有移动端问题？（手机上不好读）
- 是否可以交互？（静态 → 计算器、筛选器）

## 第四步：判断增量价值

找到缺口后，确认自己能否填补。以下形式有真实增量：

| 形式 | 示例 |
|------|------|
| 亲自验证 | 自己跑一遍任务路线，每一步截图标记 |
| 数据对比 | 版本更新前后角色的技能数值变化 |
| 交互工具 | 输入等级自动计算升级所需资源 |
| 可筛选数据 | 按职业/属性/稀有度筛选武器列表 |
| 实测对比 | 两套配装方案在实际战斗中的表现差异 |
| 解决常见错误 | 整理玩家最常遇到的 Bug 和解决方法 |

**如果你能做得更好的理由只有"我用 AI 改写了一遍"，那这不是增量价值。**

## 第五步：趋势与风险评估

使用 Google Trends 确认：

- 搜索趋势是 **上升 / 稳定 / 下降**？
- 是否受版本更新或活动影响？活动结束后需求还在吗？
- 预计需求能持续多久？

风险检查：

- 图片、Logo、地图来源是否合规？
- 是否涉及作弊、私服、赌博或侵权下载？
- 广告平台是否允许该类内容？

## 第六步：做决定

根据以上分析，决定：

- **做**：有明确的搜索意图，现有页面有明显缺口，自己能提供增量价值
- **暂缓**：有需求但增量价值不够清晰，或当前资源不足以支撑
- **放弃**：竞争过强、需求下降、风险太高或自己无法提供更好的内容
