# 第一个实验项目

> 不要一上来就想做大而全的数据库网站。从一个小而明确的主题开始，验证流程。

## 什么主题适合第一次实验

比较适合新手实验的主题通常有这些特征：

- **新游戏、新版本或新活动**带来了新问题（竞争少、需求真实）
- 用户意图明确，例如任务路线、配装、掉落位置或计算需求
- 当前搜索结果**明显过时或不完整**
- 自己能够实际游玩、测试并持续修正信息
- 可以提供截图、数据对比、计算器或交互工具

## 不适合第一次实验的主题

| 类型 | 原因 |
|------|------|
| 做了十年以上的老游戏攻略 | 竞争太强，很难突围 |
| 自己没有玩过的游戏 | 无法验证和更新内容 |
| 纯翻译其他语言的攻略 | 没有增量价值 |
| 作弊、外挂、私服相关 | 高风险，广告不支持 |
| 只有 AI 改写、没有人工验证 | Google 可能判定为低质量 |

## 第一批页面怎么做

### 数量

**5 到 10 个高质量页面。**

少比多重要。一个真正解决了用户问题的页面，胜过十个泛泛的攻略。

### 页面形式

| 形式 | 适合场景 |
|------|---------|
| 任务攻略 | 有明确步骤或路线的任务 |
| 版本数据对比 | 版本更新后数值变化的整理 |
| 计算器 | 资源计算、伤害计算、经验计算 |
| 筛选/数据库 | 按条件筛选物品、角色、武器 |
| 路线地图 | 采集路线、任务路线 |
| 进度清单 | 收集品、成就、任务的检查清单 |
| 实测对比 | 两套方案的优缺点和实测数据 |
| 常见错误 | 玩家最常遇到问题的排查和解决 |

## 实操示例

假设你选了 **"Wuthering Waves 新手攻略"** 这组关键词。

### 第 1 步：关键词细化

不要只做一个"Wuthering Waves guide"页面，这竞争太强。拆成具体问题：

| 候选关键词 | 搜索意图 |
|-----------|---------|
| Wuthering Waves beginner guide 2025 | 新手想了解基础玩法 |
| Wuthering Waves best characters tier list | 想知道哪些角色值得练 |
| Wuthering Waves echo farming route | 需要地图和路线 |
| Wuthering Waves level up materials calculator | 想算升级资源够不够 |
| Wuthering Waves all chest locations map | 收集宝箱 |

### 第 2 步：验证关键词

拿 "Wuthering Waves echo farming route" 举例：

1. **搜索意图**：玩家需要高效的资源采集路线
2. **现有结果**：YouTube 视频为主，文字攻略少，且大多数没有标记具体位置
3. **你的增量价值**：亲自跑一遍路线，截图并用箭头标注路径，提供每小时的收益对比
4. **扩展方向**：可以延伸出不同区域、不同资源类型的路线

### 第 3 步：制作页面

时间预算合理的情况下：

- 2 小时：完成任务并截图
- 1.5 小时：整理图片和编写文案
- 0.5 小时：内链设置和发布

## 迭代而非完美

你的第一个实验项目的目标**不是做一个完美的网站**，而是：

- 体验完整的 "发现需求 → 制作内容 → Google 收录 → 获得展示 → 收集数据" 闭环
- 验证自己是否愿意持续产出有增量价值的内容
- 了解从关键词到流量的实际周期和效果

不要花一个月去打磨一个完美页面。发出去、看数据、再调整。
