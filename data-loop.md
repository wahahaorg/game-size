# 数据闭环与诊断

> 网站上线只是开始。Search Console 和 GA4 告诉你——用户到底是怎么找到你的。

## Google Search Console

这是最重要的免费工具。记录：

- **哪些页面被 Google 收录了**（索引页面报告）
- **用户搜什么词找到了你**（效果 → 查询）
- **你的页面在什么位置展示**（平均排名）
- **你的页面展示了多少次**（展示次数）
- **多少人点击了你的页面**（点击率）

### 接入步骤

```txt
1. 打开 https://search.google.com/search-console
2. 添加你的网站：（两种方式任选）
   a. 域名方式：添加 DNS TXT 记录（推荐，覆盖全子域名）
   b. URL 前缀方式：上传 HTML 验证文件或添加 meta 标签
3. 提交 Sitemap（sitemap.xml 地址）
4. 等待几天到几周，开始看到数据
```

### 重点关注这 4 个指标

| 指标 | 含义 | 关注点 |
|------|------|--------|
| 索引状态 | 页面是否进入 Google 索引？ | 如果页面没被索引，不会出现在搜索中 |
| 展示次数 | Google 是否开始把页面展示给用户？ | 展示是排名的前提 |
| 实际查询词 | 用户通过什么问题找到了页面？ | 是否和你的目标关键词一致？ |
| 平均排名 | 页面目前处于什么竞争位置？ | 前 10 名才有实际流量 |
| 点击率 | 标题和描述是否值得点击？ | 排名高但没人点？说明标题不够吸引 |

### 常见问题诊断

**页面没被索引：**
- 是否提交了 Sitemap？
- 页面有没有被 `noindex` meta 标签阻止？
- 服务器有没有返回 5xx 状态码？
- 页面是不是被 robots.txt 阻止了？

**展示很少：**
- 页面是不是太新？
- 内容是否真的解决了用户问题？
- 关键词是不是太过冷门？
- 竞争页面是不是太强了？

**有展示但没有点击：**
- 标题是否清晰表达了页面价值？
- description 是否值得点击？
- 排名是否在比较靠后的位置？

## Google Analytics 4 (GA4)

> GA4 告诉你——用户来到网站后做了什么。

### 接入步骤

```txt
1. 打开 https://analytics.google.com
2. 创建新媒体资源（GA4）
3. 跟踪 ID 或代码片段嵌入网站
4. 确认数据正在收集（实时报告）
```

### 重点关注

- **用户数**：有多少独立访客
- **页面浏览量**：哪些页面被看了
- **平均停留时间**：用户是否真的在阅读内容
- **流量来源**：用户来自 Google 搜索、直接访问还是社交媒体？
- **跳出率**：用户来了是不是立刻就走了？

**特别注意：Google Analytics 的数据不能作为流量保证。它的主要用途是相对比较：哪个页面表现更好、流量来源分布如何。**

## 数据驱动决策示例

```txt
场景：你做了一个 "Wuthering Waves echo farming route" 页面

一个月后看到 Search Console 数据：
  展示次数：2,000
  平均排名：11
  点击次数：45
  点击率：2.5%

你的判断：
  ✅ 页面被索引了
  ✅ Google 开始展示它了（2,000 次展示）
  ⚠️ 平均排名 11，刚好在首页后
  ⚠️ 点击率不高，可能需要优化标题

你的行动：
  1. 优化标题：加入 "2025 最新" + 核心路线名称
  2. 补充更多区域的路线
  3. 增加内链（从其他页面链接过来）
  4. 再观察一个月
```
