# SEO 收录与排名实战

> 群友日常讨论最多的话题之一。收录速度、排名波动、怎么诊断——没有教科书，全是实战。

## 收录速度：玄学还是规律？

### 最快：秒收录

```txt
"网站秒收录了。"
— 阿飞
```

### 正常速度

```txt
"花时间做的站收录好慢啊，我给浏览器插件配的一个单页面工具，
就一个工具，别的啥文案都没，还很快收录了。"
— ZHK
```

说明：**页面复杂度与收录速度成反比**。越简单、越聚焦一个主题的页面，Google 越容易理解，越快收录。

### 收录慢的常见原因

```txt
"如果网站上线了一个月了，部分页面已请求编入索引，
但是仍然不被收录，这可能是什么原因引起的呢？
是内容质量差需要修改内容，还是需要继续发外链？"
— cyk
```

**群友给出的排查方向：**
1. 检查 Search Console 的索引报告，看报错原因
2. 确认 Sitemap 已提交且正常
3. 确认页面没有 `noindex`、`X-Robots-Tag` 之类的阻止
4. 检查 https/http/www/非www 是否做了正确的重定向
5. 内容质量：如果是内容质量导致，需要补充实质性内容

## GSC 常见诊断

### page with redirect

```txt
"我收到GSC的邮件说我有页面没被收录的原因是page with redirect，
我进去看了一下，发现我就两个页面，都是首页，
一个是https一个是http，只是从http跳转到了https，
所有http没有被收录，这个要管吗？"
— Lok

"如果你的域名选择了 https://abc.com 作为规范网址，
同时设置了 http、http://www、https://www 都重定向到 https://abc.com，
那么前三个会重定向的网址，本来就不应该被收录的。"
— 哥飞
```

### X-Robots-Tag noindex

```txt
"谷歌爬虫访问你的网站时，检测到了一个http头
X-Robots-Tag 返回了 noindex。"
— 哥飞
```

检查方向：服务器配置、CDN 配置、框架或中间件设置。

## 核心经验总结

```txt
1. 收录是排名的基础，先确认页面进了索引
2. 简单页面比复杂页面更容易被收录
3. 改动一定会影响排名，幅度决定恢复周期
4. GSC 的报错要看懂，大部分问题都有明确提示
5. 关键词密度不是排名因素，它是内容相关性的检查工具
6. 外链可以帮助恢复被影响的排名
```
