# 外链建设

> "不要等网站完美之后才开始发外链，先把外链发起来。"

## 外链的作用

群里对外链的共识：**外链是排名恢复和提升的最强杠杆。**

```txt
"为什么敢去改？我外链多，我知道怎么改，知道周期是什么。"
— 阿飞

这句话的意思是：外链多的人有底气。就算改了之后排名掉了，
也能通过外链把它拉回来。
```

## 什么时候开始发外链

```txt
"飞哥，一个没人在跑的新词，我建站几天流量也稳步上升，
这个外链什么时候开始添加比较合适？
因为我在想，一旦外链留了信息，自然被同行发现的几率要大很多。"
— BigDon🇨🇳
```

**答案不是"等完美了再发"。** 群里的做法是尽早开始，持续发布。

## 发到哪

群友用过的外链渠道：

| 渠道 | 说明 |
|------|------|
| Hacker News | 质量高，但对内容质量要求也高 |
| Reddit | 需要干净账号和干净 IP |
| 行业博客 | 如果内容够好，会被主动引用 |
| GitHub | 项目文档或 README 中引用 |
| 社交媒体 | Twitter/X、LinkedIn |
| 论坛签名 | 质量低但胜在数量 |

### Reddit 注意事项

```txt
"正常登录点赞2周的reddit帐号，今天在NoStupidQuestions评论，
被shadowban。估计vpn要用干净的才行。
我还用指纹浏览器打开的，但是没有再配一个代理。"
— 我素熊猫
```

### Hacker News

```txt
"我在hacker news发外链，说我的账号不能提交这个网站，
一般是什么原因呢？"
— 队长别开枪是我
```

## 发多少

```txt
"外链每天发。"
— 阿飞
```

不是一次性发一堆，而是**每天持续发**。质量比数量重要，但持续比爆发重要。

## 内页外链 vs 首页外链

```txt
"内页，具体小功能的页面的外链。"
— 阿飞
```

做小词/长尾词时，外链应该指向具体功能的页面（内页），而不是首页。这样 Google 才能理解这个页面是做什么的。

## 高质量外链案例

```txt
"来自群友阿强的好消息，他的网站 skillsmp.com
同时拿到了 Claude 和 Langchain 官方博客的 Dofollow 外链。
这种高质量外链，花钱都买不来的，
只能靠网站做好了，被别人主动添加。"
— 哥飞
```

**可遇不可求的外链来源：**
- 官方博客引用
- 知名工具/框架的案例页面
- 学术或教育机构引用

这些外链的价值在于：**Google 会认为你的网站被权威来源认可了。**

## 外链策略总结

```txt
1. 尽早开始，不要等完美
2. 每天持续发，不是一次性爆发
3. 小词做内页外链，大词做首页外链
4. 用干净账号和干净 IP
5. 追求优质外链 > 堆数量
6. 外链多 = 改版底气足
```
