# AI 辅助建站

> 群里很多人不是程序员，靠 AI 把网站做起来的。

## 不会写代码也能做

```txt
"有没有和我一样，不会写一行代码的同学？
就是硬着头皮搞网站，以前觉得编程离自己很远，
现在有AI，啥都敢搞。"
— H海锋
```

这是群里很大的一个人群画像：不是程序员，但借助 AI 工具完成了网站搭建、内容生成和功能开发。

## 常用 AI 工具

| 工具 | 用途 | 群友评价 |
|------|------|---------|
| Claude（Claude Code） | 全栈开发 | 稳定、代码质量高，但干一会会停下来 |
| Cursor | AI 编辑器 | "又雄起了"，适合囤特价账号 |
| Trae | AI 开发 | 适合小白，但改复杂问题时容易越改越乱 |
| ChatGPT | 内容生成、策略讨论 | "暴毙了，天天要出点幺蛾子"——不稳定 |
| GitHub Copilot | 代码补全 | 搭配 Cursor 使用 |

### 已知问题

```txt
"我已经开启了Yolo模式，但是Claude还是干一会就停下来，
```

这是目前 AI 编程工具的普遍问题：逻辑复杂或需要跨多文件修改时，AI 会需要人工介入。

## AI 辅助的安全隐患

```txt
"我今天看到V2EX里面有个人发了个自己写的新网站，
我仔细看了下他的前端js代码，发现有一段：
constructor(e='LrKgs3******') {
    this.unwatermarkApiUrl = 'https://api.unwatermark.ai',
    this.apiKey = e
}

他这个是AI写的，我估计他写完了之后，没和AI聊安全问题。"
— 星辰
```

**重要教训：** AI 生成代码可能会把 API Key、密钥等直接嵌入前端代码。使用 AI 生成代码后务必要做安全审查。

## 最佳实践总结

```txt
1. AI 能帮你搭架子、写功能，但不能替你判断需求和验证内容
2. 非程序员可以用 AI 做站，但需要学习基本的安全和部署知识
3. 提示词需要反复测试才能稳定
4. AI 写完代码后必须检查敏感信息泄露
5. Claude Code + Cursor 是目前群友比较常用的组合
6. 关键词调研和需求判断仍然是 AI 替代不了的核心能力
```
